# 🔮 Antigravity ArchiveViewer 产品交互与设计规范说明书 (PRD/Design Spec)

本规范文档面向产品经理（PM）、UI/UX 设计师以及交互设计师，系统性地定义了 **Antigravity ArchiveViewer** 的产品定位、设计美学原则、核心交互原语（UX Patterns）以及未来的商业与生态演进路线。

---

## 🎯 1. 产品核心定位与设计愿景 (Product Vision)

### 1.1 痛点与产品定位
在 AI 编程大行其道的时代，AI 助手的本地运行轨迹日志（Trajectory Logs）是一笔巨大的**“心智与代码资产”**。然而，当前行业内的日志普遍处于“文本黑盒”状态，缺乏良好的排版，且杂乱的工具调用细节导致用户极易产生信息过载与疲劳。

**Antigravity ArchiveViewer** 旨在构建一个**“懂开发、高颜值、极致流畅的开发心智大盘”**：
* **从黑盒到水晶盒**：将晦涩的开发轨迹日志封装成可随手翻阅的 Timeline 对话流。
* **从工具到看板**：通过 3 星日历将研发活跃度以极客最爱的“贡献度格子”展现，提供强烈的情感反馈和心智自豪感。
* **极致无缝嵌入**：配合 IDE 的 Simple Browser 分栏，像原生功能一样内嵌，减少窗口切换造成的专注度耗损。

## 🎨 2. 视觉设计系统与美学法则 (Design System)

为了让开发者在打开的第一眼感到惊艳（WOW 效应），ArchiveViewer 采用极具现代科技感的 **Cyber Neon 磨砂玻璃美学体系（Glassmorphism）**。

### 2.1 双主题 HSL 调色系统 (HSL Palette)
我们摒弃了普通的纯黑和白底，全部采用柔和、对齐高对比度 AAA 级可读性的 HSL 色彩模型。通过修改 HSL 变量，可实现主题一键切换与平滑渲染：

| 主题风格 | 核心色变量 (HSL) | 界面质感与视觉感知 |
| :--- | :--- | :--- |
| **赛博暗黑极客风 (Cyber Dark)** | `Primary (紫): 263 90% 64%`<br>`Background (深底): 224 25% 12%`<br>`Card (卡片): 224 25% 16%`<br>`Text (主字): 220 20% 90%` | 深邃极客夜，采用半透明卡片叠加背景微弱的霓虹发光，烘托高度专注的研发氛围。 |
| **极简护眼明亮风 (Minimalist Light)** | `Primary (紫): 263 90% 64%`<br>`Background (纯白): 210 20% 98%`<br>`Card (卡片): 0 0% 100%`<br>`Text (主字): 224 25% 12%` | 极简北欧风，利用极轻的冷灰阴影拉开视觉层级，确保在强光环境下阅读依然舒适、不刺眼。 |

### 2.2 磨砂玻璃层级规范 (Glassmorphism & Depth)
为了突出前沿的产品格调，主框架和对话卡片使用了磨砂玻璃（Glassmorphism）层级：
* **背景模糊**：卡片与侧边栏的边框和背景必须使用 `backdrop-filter: blur(16px)`。
* **极细边框**：边框厚度限制在 `1px`，在暗黑主题下使用 `rgba(255,255,255,0.06)`，在明亮主题下使用 `rgba(0,0,0,0.08)`，模拟出精致的玻璃微边缘反射。
* **悬浮阴影**：卡片悬浮（Hover）时，使用 `transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1)`，阴影从平面微过渡到向外延伸的霓虹柔光，赋予界面动态呼吸感。

## 🧩 3. 核心交互原语与动效设计 (UX Patterns)

交互细节是区别“平庸工具”与“殿堂级产品”的试金石。

### 3.1 3星强度圆点网格日历 (Calendar Activity Grid)
* **设计意图**：致敬 GitHub 的 Contribution Grid，但采用更灵动、更内敛的 3 星圆点等级系统。
* **星级指示器（Dots Container）**：
  在每日日期格子下方分配 3 个微型 Dot 容器。圆点分为“实心 `●`”和“空心 `○`”，结合交互轮数（Turn Count）按如下比例点亮：
  * 轻度交互（1-4 轮）：`● ○ ○`（单点燃亮，代表今天完成了常规修改）
  * 中度交互（5-12 轮）：`● ● ○`（双点燃亮，代表经历了中等强度的功能迭代）
  * 重度交互（13 轮以上）：`● ● ●`（三点全亮，代表深度结对编程或经历了高难度的排错，今日成就感拉满！）
* **交互逻辑**：点击日期锁定过滤当日会话列表，下方自动升起优雅的“🧹 清除日期过滤”按钮。

### 3.2 智能长文本可视限高折叠 (Collapsible Mask)
* **体验痛点**：若代码块过长，用户需要不断滚动屏幕才能跳到下一轮对话，这极其割裂心智。
* **交互规范**：
  * **高度警戒线**：`260px`。当文本/代码编译后的真实高度超出 `260px` 时，组件强行将高度钳制在 `250px`。
  * **毛玻璃遮盖**：限高高度底部 **`60px`** 范围内，强制加盖一层渐变毛玻璃半透明遮罩（`linear-gradient(transparent, var(--bg-card))`），暗示下方仍有内容。
  * **平滑拉伸动效**：点击“展开余下全文”，最大物理限高（`maxHeight`）从 `250px` 平滑过渡至内容实际高度，严禁出现突兀的视觉跳跃。

### 3.3 双向拖拽侧边栏分割线 (Resizer)
* **交互规范**：
  * 侧边栏与主窗口间保留一条 4px 细线。当鼠标悬浮在其上时，指针瞬间变为 `col-resize`，暗示可拖拽。
  * 拖拽过程中，利用 React 和全局 mouse 事件锁死浏览器文字选中，避免界面出现蓝色选中色块。
  * 侧边栏宽度强行限制在 **`230px`**（保证日历和列表不挤压）至 **`480px`**（保证 Timeline 有足够显示空间）之间。

### 3.4 搜索平滑滚动定位与紫色霓虹脉冲 (Smooth Scroll & Pulse Glow)
* **设计意图**：打破传统的“跳转直接切页”粗暴体验。
* **交互逻辑**：
  1. 用户在“超级全局搜索”中点击某条历史匹配步骤。
  2. 界面秒级无缝返回 Timeline 阅读大盘。
  3. 目标卡片平滑居中滚动对齐（`scrollIntoView({ behavior: 'smooth', block: 'center' })`）。
  4. 滚动到位后，卡片外围发射出**紫色霓虹呼吸灯脉冲**（Pulse Glow），动画时长为 **`2500ms`**，让光圈缓缓向外扩散并彻底淡出，精准捕捉用户的视觉焦点。

## 📈 4. 未来产品演进与商业潜力 (Product Roadmap)

ArchiveViewer 极具扩展性，具备成长为独立高价值开发工具或被其他 AI Assistant 厂商深度集成的巨大潜力。

### 4.1 近期功能优化 (Near-term Features)
1. **一键智能导出与分享**：
   支持将精美的 Timeline 对话流一键导出为带有内联样式的独立 HTML、PDF 或标准 Markdown，方便开发者在团队内部分享解题思路。
2. **Token 与成本消耗仪表盘**：
   后端自动解析 transcript.jsonl 中记录的 prompt_tokens 与 completion_tokens。前端集成 Chart.js，绘制日/周/月的 Token 成本花费曲线与 AI 交互效率曲线。

### 4.2 中期商业化构想 (Mid-term Commercial Value)
1. **轻量级 Git Diff 可视化编辑器**：
   在工具调用 console drawers 中，将原本冷冰冰的控制台返回文本，替换为并排对比的物理代码 Diff 视图（可嵌入轻量级 Monaco Editor），让 AI 的每一步文件写操作都一目了然。
2. **多终端多平台分发 (Tauri/Electron Wrapping)**：
   将当前的静态前端与原生 Node 后端打包为独立的 macOS/Windows 桌面 App 客户端（利用 Tauri 构建，安装包体积小于 10MB）。支持常驻系统 Menu bar 菜单栏托盘，开机自动运行。
3. **AI 厂商生态化集成**：
   当前各类 AI 命令行与编辑器插件（如 Claude Code, Aider, Cline）百花齐放，但均缺乏精美的历史大盘。ArchiveViewer 的数据接口和前端布局可作为通用的可视化规范，授权或开源集成进各大家族的 AI Tool 中，具备极高的话语权与商业推广空间。
