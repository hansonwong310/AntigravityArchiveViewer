# 🔮 Antigravity ArchiveViewer

> **开箱即用、极致交互的 Antigravity 历史会话本地可视化大盘**

---

## 📖 1. 项目简介与核心痛点解决

**Antigravity ArchiveViewer** 是一款专门为智能 AI 编程助手 **Antigravity** 打造的本地轨迹日志（Trajectory Logs）图形化大盘工具。

在日常使用 AI 结对编程的过程中，AI 助手会在底层生成保真度极高、记录全部工具调用与决策链路的日志文件（`transcript.jsonl`）。然而，直接阅读这些 JSONL 原始文本非常困难，更难以在海量的历史会话中迅速召回之前的修改细节。

**Antigravity ArchiveViewer** 旨在解决这些核心痛点，提供以下核心价值：
* **解密 AI 决策黑盒**：将晦涩的 JSONL 转换为清晰美观的 **ChatGPT 风格标准对话气泡流**。
* **开发过程可溯源**：提供**全局全文 AND 检索**，可深度穿透至历史日志以及工具执行的 Console 返回值中，一键秒级召回。
* **工作活跃度看板**：直观的 **3星工作强度日历圆点网格**，一眼感知过往交互频次，将隐形的工作量完全可视化。
* **开发环境无缝融合**：经过深度排版优化，**支持在 VS Code / Cursor 中右侧分栏无缝嵌入**，真正做到“左边写代码，右边看历史”。

## 📦 3. 快速上手

ArchiveViewer 采用**零外部依赖（0-Dependency）**架构设计。服务端使用原生 Node.js，前端采用 CDN 形式的 React 18 与 Babel，无需任何 `npm install` 或者是 `npm run build`，冷启动仅需 **50 毫秒**。

### 第一步：启动本地服务端

在项目根目录下，我们提供了一个优雅的彩色日志启动脚本 `AntigravityAV.sh`。

运行以下命令启动服务：
```bash
./AntigravityAV.sh
```

> **💡 启动环境检测逻辑**：
> 1. 脚本会优先定位 Antigravity 专用的极速运行时 `agy-node`。
> 2. 若在环境或默认安装路径中未检测到 `agy-node`，会平滑降级使用系统自带的 `node` 运行时。

当终端输出如下内容时，代表服务端已成功就绪：
```text
📡 Antigravity ArchiveViewer 服务端已启动！
🔗 访问地址: http://localhost:5173
🧠 归档数据目录: /Users/hansonwang/.gemini/antigravity/brain
```

### 第二步：在 IDE (VS Code / Cursor) 中内嵌使用

为了获得最完美的边开发边查阅的体验，建议将 ArchiveViewer 内嵌在编辑器分栏中：

1. 在 IDE 中按下快捷键：`Cmd + Shift + P` 打开命令面板。
2. 输入并选择：`Simple Browser: Show` (内置极简浏览器)。
3. 在地址栏中输入：`http://localhost:5173` 并回车。
4. 页面载入后，**将该 Simple Browser 的网页标签向右侧拖动进行分栏**。
5. 完成！现在您可以实现左侧编写代码，右侧实时翻阅和深度检索历史轨迹的极客开发流。

## ⚙️ 配置与数据路径说明

* **默认服务端口**：`5173`（可在根目录 `server.js` 中修改 `PORT` 变量）。
* **日志基准路径**：`~/.gemini/antigravity/brain`
  * 服务端会自动读取该路径下所有以 UUID 命名的子目录，并流式解析里面的 `.system_generated/logs/transcript.jsonl` 日志。
  * 若该路径不存在，服务端会自动返回空列表，不会引起程序崩溃。
* **CORS 跨域支持**：后端已默认配置 `Access-Control-Allow-Origin: *`，方便前端或第三方插件进行 API 调试与集成。

## 💡 日常使用小贴士

1. **精准定位**：如果您想在庞大的日志中定位某个具体的 shell 指令，在“超级全局搜索”中输入 `run_command` 加上您的指令关键词即可，例如 `run_command git`。
2. **日历网格的星级指标**：
   * `● ○ ○` 轻度交互 (1-4 轮对话)
   * `● ● ○` 中度交互 (5-12 轮对话)
   * `● ● ●` 重度交互 (13 轮及以上对话，代表您今天经历了深度开发或排错！)
3. **数据自动更新**：在左侧搜索栏下方，可以将自动刷新频率设为“每 5 秒”或“每 15 秒”。这样在您与左侧 AI 编程助手对话的同时，右侧的大盘会自动吸纳最新产生的日志。
